Terraform Introduction
======================

Ansible can create, manage, and update infrastructure, but it is not a stateful tool like Terraform.
Ansible contains Collections/Modules. It will have parameters
Ansible can create infrastructure but it cannot manage and update infra because of no state management feature available in ansible
Ansible originally created for configuration management and not good for infra management

Interview:-
Ansible is a stateless configuration management tool that uses modules from collections to manage systems in an idempotent way.
It can create and manage infrastructure, but since it does not maintain a state file, it is not ideal for complex infrastructure lifecycle management.
Therefore, Terraform is typically used for infrastructure provisioning, and Ansible is used for configuring the created servers.

Provider:-
--------
A provider in Terraform is a plugin that allows Terraform to interact with a specific platform like AWS, Azure, or GCP.

provider "aws" {
  region = "us-east-1"
}

provider "azurerm" {
  features {}
}

provider "google" {
  project = "my-project"
}

AWS CLI is one way to configure authentication, but actual authentication is done using AWS credentials (Access Key & Secret Key).
aws configure
The aws configure command is used to set up AWS credentials by providing the Access Key, Secret Key, default region, and output format.
